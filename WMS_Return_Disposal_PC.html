概要：返品品の廃棄申請を一覧管理する画面。倉庫スタッフは申請状況を確認し、荷主ユーザーは承認待ちの案件を把握できる。タブ切り替えで「承認待ち」「承認済み」「却下」を確認。
仕様：disposal_approvals テーブル参照。status: pending（承認待ち）/ approved（承認済み）/ rejected（却下）。廃棄起点は常に荷主指示（QA-5③）。

W-D002廃棄申請登録 倉庫スタッフ
概要：倉庫スタッフが返品検品後に廃棄が必要と判断した品目を申請登録する画面。申請後、荷主ユーザーに通知され承認待ち（pending）状態に遷移する。
仕様：disposal_approvals.requested_by（倉庫スタッフ）/ disposition_note は必須入力（QA-5）/ 廃棄品は quarantine ステータスのみ対象（出荷許可マトリクス）。

廃棄理由（disposition_note）
目的区分=廃棄（purpose_type='disposal'）の場合、disposition_note は必須入力です（QA-5 設計）。荷主への報告根拠となります。
申請後の流れ
1. 廃棄申請を送信（disposal_approvals.status = 'pending'） →  2. 荷主ユーザーに通知 →  3. 荷主が承認（status = 'approved'）または却下（status = 'rejected'） →  4. 承認後、廃棄出荷処理を実行（shipment_orders.purpose_type = 'disposal'） →  5. 廃棄完了記録（WMS DB のみ・廃棄証憑は荷主側で管理）
W-D003荷主承認画面 荷主ユーザー専用
概要：荷主ユーザーが廃棄申請の詳細を確認し、承認または却下を行う画面。廃棄は荷主の法的所有権に基づき荷主の最終意思確認が必須（Q10-5 C案確定）。承認後に廃棄出荷処理（purpose_type='disposal'）が実行可能になる。
仕様：disposal_approvals.approved_by（荷主ユーザー）/ shipper ロール権限必須 / 承認時に approved_at セット。
