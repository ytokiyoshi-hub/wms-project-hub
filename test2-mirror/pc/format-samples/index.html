📋 トップ / フォーマット見本 6種類 時吉さん順次確認用
マルキ食品 WMS で使う 6種類のフォーマット見本 一覧。
時吉さんに順次確認していただき、各フォーマットの確定後に全画面 (マスタ130・監査16・統合14・在庫/出荷/入荷系) を該当フォーマットに一気展開する。
凡例: 確定 = 時吉さん許容済 / NEW = 今回新規作成 / 既存 = 既存見本流用
① 一覧フォーマット 確定
2ペイン構造 (左=一覧テーブル / 右=業務情報320px sticky)。
用途: マスタ・実績・履歴・監査ログ等の「行を眺める」画面。layout_c と同じ構造で products.html が見本。
products.html (マスタ見本) layout_c.html (在庫見本)
② 詳細フォーマット 既存
親画面 (一覧) から1件選んで開く子画面。ヘッダ情報 + 明細テーブル + 進捗ストリップ。
用途: 入荷予定詳細・出荷予定詳細・受注詳細・ロット詳細 等。
plan_detail.html (W-401-D 子画面) plan_line_detail.html (孫画面)
③ 登録/編集フォーム NEW
2ペイン構造 (左=form-row入力フィールド+保存ボタン / 右=入力中サマリ+関連情報+直近の入力履歴)。
用途: 入荷予定登録・受注登録・マスタ新規追加 等。フィルタ画面の filter-row とは別物。
form_sample.html (見本) register.html (W-402 既存)
④ ダッシュボード / KPI NEW
上=KPIカード横並び (6枚) / 中=ステータステーブル (3枚) / 下=本日のアラート/要対応。
用途: トップダッシュ・拠点別ダッシュ・荷主向けダッシュ・工程管理ダッシュ 等。見せ方B (普段グレー・要対応のみ accent/alert)。
dashboard_sample.html (見本) dashboard.html (既存)
⑤ HT画面 既存
ハンディターミナル (BT-A2000 / 4インチ) 専用UI。実機相当のスクリーンモック + F1-F4 ファンクションキー。
用途: 検品・棚入れ・ピッキング・棚卸 等の現場作業。/ht/ 配下で別CSS (ht-mock.css)。
ht/inspect/asn.html (HT-403-1) ht/menu.html (HT メニュー)
⑥ 作業手順画面 既存
HT作業の流れを順番にこなす型 (1画面=1ステップ)。前画面→次画面のジャンプリンクで業務貫通。
用途: 検品 (ASN選択 → スキャン → 完了) / ピッキング (ウェーブ → ロケ → 数量 → 完了) / 棚入れ (指示 → ロケ → 完了)。
ht/pick/scan.html (ピック2/3) ht/inspect/scan.html (検品2/3) ht/putaway/instructions.html (棚入れ1/3)
