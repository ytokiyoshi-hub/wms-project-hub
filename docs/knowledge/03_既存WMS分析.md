# 03 既存WMS分析

**カテゴリ：自社WMSの機能分析・競合比較**  
**最終更新：2026-05-07**  
**ステータス：骨格ドラフト**

---

## このドキュメントの目的

私たちが開発しているWMS（v3.1）の機能を体系的に分析し、強み・特徴・差別化ポイントを整理する。  
「なぜこの機能が必要か」「何が独自なのか」をチーム全員が説明できる状態を目指す。

---

## 1. WMSの基本スペック

| 項目 | 内容 |
|------|------|
| **システム名** | WMS 3PL対応版 |
| **バージョン** | v3.1（仕様策定完了）|
| **対象規模** | 中〜大規模3PL倉庫 |

## 2. 機能カテゴリ別概要（136機能）

### A. 出庫機能（1件）
出荷指示の新規登録。用途区分（配送先・廃棄・返品等）をすべて「出荷」として処理。  
→ 廃棄・サンプル等を別処理にしない設計方針が重要（詳細は[07_経験談.md](07_経験談.md)）

### B. 在庫ステータス管理（4件）
- ステータスマスタ（通常/出荷止め/引当除外等）
- ロット単位の出荷止め機能
- ロケーション単位の引当除外
- ステータス遷移履歴（変更記録を保持）

### C. ピッキングエリア・並行作業（7件）
**差別化ポイント**：エリア並行方式とゾーンリレー方式を**受注ごとに選択**できる。

| 方式 | 概要 | 向いているケース |
|------|------|----------------|
| エリア並行 | 複数作業者が同時進行 | 大量・単品中心 |
| ゾーンリレー | エリアを順番にリレー | 多品目・少量 |

### D. 引き渡し・積込工程（6件）
積込管理・積込明細書・HT積込確認・集荷時刻アラート・配送業者マスタ・引渡し記録。  
「積んだが記録がない」「引渡しの証跡がない」という現場課題を解決。

### F. 仮置き工程（5件）
カゴID管理で「どのカゴがどの便に乗るか」を追跡。仮置きエリアのロケーション管理。

### G. 出荷実績送信（7件）
荷主への実績データ自動送信。業界標準フォーマット対応。送信エラー検知・アラート。

### H. 3PL対応・マルチテナント（11件）
**最重要カテゴリ**。複数荷主を1倉庫で受託する3PL事業の核心。

| 機能 | 概要 |
|------|------|
| 荷主マスタ強化 | 荷主ごとの詳細設定 |
| データの荷主別分離 | 完全なマルチテナント実装 |
| ユーザー荷主アクセス権 | スタッフごとに見える荷主を制限 |
| 荷主切替機能 | 作業中に荷主を切り替え |
| 荷主専用/共用ロケ管理 | 専用エリアと共用エリアの混在対応 |
| 料金計算・請求書発行 | 荷主別の保管料・荷役料計算 |
| 荷主別KPIレポート | 荷主ごとの業績可視化 |
| 荷主ポータル | 荷主自身が在庫・出荷を確認 |

### I. 機能ON/OFF設定（7件）
**差別化ポイント**：機能ごとに有効/無効を切り替えることで、1つのシステムで複数の業態・倉庫規模に対応。

```
機能ON/OFFの依存チェック機能も実装：
例）ピッキングエリア機能をOFFにすると、
  → ゾーンリレー機能も自動的に無効化される
```

### J. カレンダー管理（4件）
倉庫カレンダー / 配送業者カレンダー（休日の便自動振替）/ 荷主カレンダー（割増課金） / 緊急対応フラグ

### K. 人員・シフト管理（16件）
**差別化ポイント**：WMSにシフト管理・労務管理を統合。生産性11指標を自動計算。

主要機能：
- 物量予測 → 必要人員自動算出 → シフト提案（11項目）
- 荷主別労務コスト算出（どの荷主の仕事に何人・何時間使ったか）
- 従業員スマホアプリ（シフト確認・業態実績入力）
- 法令対応支援（労働基準法遵守チェック）

### M. 外部倉庫対応・課金詳細（13件）
3PL事業者が別の倉庫（外部委託先）を使う「再委託（再保）」に対応。  
外部倉庫への支払単価・自動計算・委託先請求書との突合機能。

### N. 請求書明細・根拠提示（11件）
「なぜこの金額になったのか」を荷主が自分で確認できるレベルの明細提示。  
自動生成 / PDF出力 / CSV出力 / 前月比較・差異説明。

### O. 収支・生産性レポート（7件）
倉庫事業者向けの経営レポート。荷主別ランキング / 季節変動分析 / 委託先利用の影響分析。

### P. ロケ種別による引当・補充最適化（11件・新規）
ロケーション種別（パレット/棚/流動棚）に基づく引当ロジック。  
出荷実績から自動的に補充閾値を計算し、季節変動にも追従。

### Q. 配送業者の自動振り分け（8件・新規）
配送先住所から方面（エリア）を自動判定 → 便を自動選択 → 便容量上限で自動振替。  
従来の手動振り分け作業を大幅削減。

### R. 商品コード体系（4件・新規）
荷主コードとWMS内部枝番を管理。リネームせず枝番追加で履歴を保持（欠番なし運用）。

### S. 梱包ロジック・梱包材管理（11件・新規）
梱包材マスタ + 商品別梱包係数 + 引当時の個口数自動確定。  
梱包資材の在庫管理・コスト算出まで統合。

### T. ユーザー権限管理（4件・仮）
時給・労務コスト情報へのアクセス制御。ロール別権限（詳細は後日）。

## 3. 設計確定事項（業務ルール）

| ID | 設計決定 | 理由 |
|----|---------|------|
| E-1 | 受注ごとに引当方式を選択 | 出荷内容によって最適な方式が異なる |
| E-2 | 作業者は固定の専門担当を持たない | 柔軟なシフト配置のため |
| E-3 | 日次でリーダーが柔軟に担当エリア割振 | 物量変動への対応力 |
| E-4 | 3PL運用（複数荷主受託） | ビジネスモデルの核心 |
| E-5 | 機能ON/OFF制採用 | 1システムで複数業態対応 |
| E-6 | ピッキングはFIFO強制 | 食品・消費期限管理の必須要件 |

## 4. 競合WMSとの差別化ポイント

### 一般的なパッケージWMSにない機能

| 機能 | 一般WMS | 自社WMS |
|------|---------|---------|
| マルチテナント（複数荷主） | 対応少ない | ✅ 完全対応 |
| シフト管理の統合 | 別システム | ✅ WMS内で完結 |
| 荷主別労務コスト | 非対応が多い | ✅ 対応 |
| 機能ON/OFF | 対応少ない | ✅ 機能単位で設定 |
| 梱包材コスト追跡 | 対応少ない | ✅ 対応 |
| 配送業者自動振り分け | 対応少ない | ✅ 対応 |

### 弱みと課題

- 現時点は仕様策定フェーズ（実装未着手）
- カスタマイズ性と標準化のバランスが今後の課題
- ドキュメントの多言語対応（グローバル展開時）
