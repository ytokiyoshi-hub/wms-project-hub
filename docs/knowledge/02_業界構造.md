## 1. 物流の全体像

### 1PL〜4PLの区分

| 区分 | 説明 | 例 |
|------|------|-----|
| **1PL**（自社物流） | 荷主が自社で物流を運営 | 大手製造業の自社倉庫・自社トラック |
| **2PL**（輸送特化） | 輸送のみを外部委託 | 一般的な運送会社（ヤマト・佐川等） |
| **3PL**（包括委託） | 保管・荷役・輸送・情報管理を一括委託 | **私たちのターゲット市場** |
| **4PL**（管理者型） | 複数の3PLを束ねるコーディネーター | サプライチェーン全体最適化 |

### 私たちのWMSが対象とする3PL

> **3PL（サード・パーティー・ロジスティクス）**  
> 荷主が物流業務を第三者（3PL事業者）に包括的に委託する形態。  
> 3PL事業者は「倉庫の運営」から「出荷管理」「荷主への報告」まで代行する。

## 2. 登場人物（ステークホルダー）

### 3PL取引における関係図

```
[荷主（Shipper）]
  │ 出荷指示・在庫照会
  ▼
[3PL事業者（倉庫オペレーター）]  ← 私たちのWMSを使う人
  │ 作業指示
  ├── [倉庫作業員] ← HTを使う人
  │
  ├── 配送委託
  ▼
[配送業者（キャリア）]
  │
  ▼
[配送先（エンドユーザー・店舗等）]
```

### 各ステークホルダーの関心事

| 立場 | 主な関心事 |
|------|-----------|
| **荷主** | 在庫が正確に管理されているか / 出荷が時間通りに行われるか / 請求金額が正しいか |
| **3PL事業者** | 作業効率 / 人件費管理 / 複数荷主の同時管理 / 請求業務の正確さ |
| **倉庫作業員** | 作業指示のわかりやすさ / HTの操作性 / ミスが起きにくいシステム |
| **配送業者** | 集荷時刻の遵守 / 積込の正確さ |

## 3. 3PL市場の現状（日本）

### 市場規模と成長

- 日本の3PL市場は2009年から15年間で**約3倍以上に拡大**
- EC（ネット通販）の急拡大が最大の成長ドライバー
- 主な拡大要因：ECの小口多頻度配送、荷主の物流アウトソーシング推進

### 現在の主要課題（2024-2025年）

| 課題 | 詳細 |
|------|------|
| **人手不足** | 少子化による労働人口減少。倉庫作業員・ドライバー共に不足深刻 |
| **2024年問題** | トラックドライバーの時間外労働規制。輸送力が最大14%低下とも試算 |
| **積載率低下** | 営業用トラックの積載効率が39.7%（2022年度）まで低下 |
| **EC小口化** | 配送件数は増加しているが1件あたりの量が少ない → コスト増 |
| **コスト上昇** | 燃料費・人件費・システム投資費用の増加 |
| **DX遅れ** | 中小物流企業のデジタル化が遅れており、手作業・紙管理が残存 |

## 4. 倉庫業の構造

### 倉庫の種類

| 種別 | 特徴 |
|------|------|
| **自社倉庫** | 荷主が所有・運営。コスト固定だが管理しやすい |
| **営業倉庫** | 外部の倉庫業者に寄託。倉庫法に基づく契約 |
| **外部委託（再保）** | 3PL事業者がさらに別の倉庫に再委託 |

### 倉庫業法の重要ポイント

**倉庫業者の立場**：荷主の寄託物を保管する立場であり、**荷主の指示なしに在庫を処分することは法律・契約上不可**。

→ これが「その他出庫（廃棄・サンプル等）」をWMSで独立した機能として持たない理由。廃棄もサンプルも全て荷主の指示（出荷指示）として処理する。

## 5. 物流コストの構造

### 荷主が負担する物流コスト

```
物流コスト合計
├── 保管料（倉庫使用料）
│   ├── 3期制：月3回の在庫量で計算
│   ├── 2期制：月2回の在庫量で計算
│   └── 坪貸し：確保面積×単価（固定）
├── 荷役料（入庫・出庫・ピッキング等の作業料）
├── 配送料（輸送費）
└── その他（システム費・管理費）
```

### 3PL事業者の収益構造

- **収入**：荷主からの保管料・荷役料・システム利用料
- **原価**：倉庫賃料・人件費・システム費・外部委託費（再保）
- **差益**：倉庫の運営効率・作業生産性が利益に直結

## 6. WMSの市場ポジション

### 主要プレイヤー

| タイプ | 特徴 |
|--------|------|
| **パッケージWMS** | 汎用的・低コスト。小〜中規模向け |
| **スクラッチ開発** | カスタマイズ自由。大規模・特殊業態向け |
| **クラウドWMS** | 導入が早い・運用コスト低。中小向けに普及中 |
| **ERP内臓WMS** | SAPなどERP付随。大手製造業・流通向け |

### 私たちのWMSの立ち位置

- **3PL専業の中〜大規模倉庫向け**
- 複数荷主対応（マルチテナント）が核心機能
- 機能ON/OFFで幅広い業態に対応
- 食品卸BtoB+EC（複数温度帯・複数荷主）をベースシナリオとして設計
