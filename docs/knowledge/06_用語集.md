# 06 用語集

**カテゴリ：物流・WMS用語の定義集**  
**最終更新：2026-05-07**  
**ステータス：骨格ドラフト**

---

## このドキュメントの目的

WMS・物流業界で使われる専門用語を統一定義する。  
「この言葉、どういう意味だっけ？」を解消し、チーム内のコミュニケーション齟齬を防ぐ。

> **注意**：このプロジェクトでは倉庫業界の正式用語を使用する。誤った表現は使わないこと。

---

## セクション1：業界用語（正式表現の確認）

### ⚠️ よく間違える用語対照表

| ❌ 使わない表現 | ✅ 正しい表現 | 補足 |
|--------------|------------|------|
| 受注 | **出荷指示 / 出荷** | 倉庫業者は「受注を受ける」のではなく「出荷指示を受ける」 |
| 課金 | **請求 / 料金**（文脈による） | 「課金する」→「請求する」「料金を設定する」 |
| もらう / もらわない | **収受 / 控除** | 料金の授受を指す正式表現 |
| 在庫数を合わせる | **棚卸し差異を調整する** | |
| 商品を捨てる | **廃棄処理（荷主指示）** | 倉庫業者の意思では処分不可 |

## セクション2：倉庫・在庫管理の基本用語

### 物流・倉庫業

**3PL（サード・パーティー・ロジスティクス）**  
荷主が物流業務（保管・荷役・輸送・情報管理）を第三者（3PL事業者）に包括委託する形態。

**荷主（にぬし）**  
倉庫に商品を預ける企業。WMSでいう「テナント」に相当。  
本プロジェクトのベースシナリオ企業：株式会社マルキ食品（架空）。

**倉庫業者 / 3PL事業者**  
荷主の商品を預かり、保管・入出庫・配送手配を行う企業。WMSを使う側。

**寄託（きたく）**  
荷主が倉庫業者に物品の保管を委託すること。倉庫業法に基づく法的関係。

**再保（さいほ）/ 外部委託**  
3PL事業者が、さらに別の倉庫・物流業者に業務を再委託すること。

**HT（ハンディターミナル）**  
倉庫現場で使うバーコード読み取り機能付き端末。本プロジェクト対応機種：  
キーエンス BT-A2000（4インチ）/ BT-A1000（3インチ）

---

### ロケーション管理

**ロケーション**  
倉庫内の棚・置き場の位置を管理する単位。商品は「どのロケーションに何個あるか」で管理。

**ロケーションコード**  
```
形式：{階}{フロア区分}{種別英字}-{通路2桁}-{列}{段}{位置}
例：3AP-02-134（3階Aフロア・パレットロケ-02通路-1列3段4位置）
```

**ロケーション種別**  
| 英字 | 種別 | 用途 |
|------|------|------|
| P | パレット | 大量保管。フォークリフト対応 |
| T | 棚（バラ） | ピース単位のピッキング |
| F | 流動棚 | 先入れ先出し専用棚 |
| K | 仮置き | 一時保管。梱包済み荷物の集積 |
| R | 不適合品 | クレーム品・破損品 |
| H | ハイラック | 高所棚。特殊フォーク必要 |

**フリーロケーション**  
入庫のたびに空いているロケーションに格納する方式（固定ロケの逆）。  
スペース効率が高いが、在庫管理がシステム依存になる。

---

### 在庫管理

**FIFO（First In First Out）**  
先入れ先出し。早く入庫した商品から先に出荷する原則。  
食品・消費期限管理に必須。本プロジェクトでは強制適用（E-6）。

**ロット**  
同一の製造条件で生産された商品のまとまり。  
食品ではロット＝製造日・消費期限の管理に使う。

**ロット出荷止め**  
問題のあるロットを出荷禁止にする機能。食品の異物混入・品質不良発覚時に使用。

**在庫引当（ざいこひきあて）**  
出荷指示に対して在庫をピッキング対象として確保すること。  
引当後は他の出荷指示からは見えなくなる。

**補充（リプレニッシュメント）**  
ピッキングロケーションの在庫が少なくなったとき、  
バックストック（パレットロケ等）から補充すること。

**棚卸し（たなおろし）**  
実際に倉庫を確認して在庫数を数え、WMSの記録と照合すること。  
定期棚卸し（月次・年次）と不定期棚卸し（差異発生時）がある。

---

### 出荷・配送

**ウェーブ（Wave）**  
複数の出荷指示を一括処理するための作業単位。  
配送便の締め時刻・商品カテゴリ・温度帯などでウェーブを分割する。

**ピッキング**  
出荷指示に基づき、倉庫の棚から商品を取り出す作業。

**梱包（こんぽう）**  
ピッキングした商品を段ボール等の梱包材に詰める作業。  
梱包材の選定・個口数（箱数）の確定を行う。

**個口数（こぐちすう）**  
1件の出荷で必要な箱・パレットの数。梱包確定時に確定する。

**荷札（にふだ）**  
配送先・送り状番号等を印刷したラベル。梱包確定後に発行。

**方面（ほうめん）**  
配送先の地域区分。「関東方面」「近畿方面」など。  
配送業者の自動振り分けに使用（Q-1「方面マスタ」）。

**積込（つみこみ）**  
梱包済みの荷物を配送トラックに載せる作業。

**引き渡し記録**  
配送業者に荷物を引き渡したことの記録。荷主への実績報告・トラブル時の証跡。

## セクション3：3PL・マルチテナント用語

**マルチテナント**  
1つのWMSシステムで複数の荷主（テナント）を管理する仕組み。  
データは荷主別に完全分離される。

**荷主別分離**  
A社の在庫・B社の在庫が絶対に混在しないようにするシステム設計。  
権限管理・データベース設計の両面で担保する。

**荷主専用ロケ**  
特定の荷主だけが使えるロケーション。他荷主の商品は格納できない。

**共用ロケ**  
複数荷主の商品が混在するロケーション。システムでどの荷主の商品かを管理。

## セクション4：請求・料金用語

**保管料（ほかんりょう）**  
在庫を倉庫に預けることへの対価。計算方式は以下の3種類：

| 方式 | 計算式 |
|------|--------|
| 3期制 | (前月繰越残 + 10日残 + 20日残 + 入庫総数) × 単価 |
| 2期制 | (前月繰越残 + 15日残 + 入庫総数) × 単価 |
| 坪貸し | 確保面積 × 単価（固定） |

**荷役料（にやくりょう）**  
入庫・出庫・ピッキング等の作業に対する対価。  
件数・ケース数・時間などを単位として計算。

**期分け（きわけ）**  
保管料計算のための「期末日」の設定。  
3期制なら月を3分割（10日・20日・末日）に区切る。

**控除（こうじょ）**  
再委託（外部倉庫使用）のコストを荷主請求から差し引くこと。

## セクション5：商品・マスタコード体系

**マスタコード形式（統一）**  
すべて「英2桁＋3桁数字」：

| マスタ | プレフィックス | 例 |
|-------|--------------|-----|
| 倉庫（自社） | WH | WH001 |
| 倉庫（外部委託） | YU / TK等 | YU001 |
| 荷主 | MK | MK001 |
| ピッキングエリア | AR | AR001 |
| 配送業者 | CR | CR001 |
| 取引先 | TP | TP001 |

**枝番（えだばん）**  
同一商品の微差（規格違い・バージョン違い等）を管理する番号。  
通常は枝番なし（S-00321）、必要時のみ-02から追加（S-00321-02）。  
枝番はリネームせず保持（履歴重視）。

**商品単位の4階層**  
パレット > ケース > ボール > ピース  
商品マスタには「1段下の単位」で保持（例：ケースには「1ケース = N ボール」を持つ）。
