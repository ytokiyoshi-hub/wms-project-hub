```javascript
function classify(title) {
  if (/CSV|PDF|DL|ダウンロード|エクスポート|出力|印刷|発行/.test(title)) return "output";
  if (/詳細|プレビュー/.test(title))                                      return "detail";
  if (/編集|変更|修正|更新/.test(title))                                  return "edit";
  if (/新規|追加|登録|作成|起票/.test(title))                              return "create";
  if (/削除|失効|廃棄/.test(title))                                       return "delete";
  if (/テスト|監視|調査|確認|シミュレーション|再生/.test(title))             return "test";
  if (/実行|送信|承認|確定|開始|発動|連携|消込|統合|再送|招待|割当/.test(title)) return "exec";
  return "generic";
}
```

## 5. エッジケース検証

### 5-1. 特殊文字タイトル（XSS安全性）

全タイトルは showModal 内で `esc()` 関数により HTML エスケープされる。

| エッジケース | 結果 |
|------------|------|
| ダブルクォートを含むタイトル | `&quot;` にエスケープ → ✅ safe |
| 改行を含むタイトル | テキストとして表示（`\n` のまま body に入るが XSS にはならない） → ✅ safe |
| SQL injection ライクな文字列 | `&#39;; DROP TABLE todos;--` にエスケープ → ✅ safe |
| 矢印記号（→）を含むタイトル | そのまま表示 → ✅ safe |
| 絵文字を含むタイトル（📷 添付） | そのまま表示 → ✅ safe |

実際の対象HTMLに特殊文字を含むタイトルは **0件**（全451ボタン調査済）。

### 5-2. テーブル外ボタン（rowData なし）

| ケース | 動作 |
|--------|------|
| create ボタンが table 外 | 常に空フォーム（行データ不要）→ ✅ 変化なし |
| edit ボタンが table 外 | pre-fill なし（空 input フォーム）→ ✅ 安全（機能低下のみ） |
| delete ボタンが table 外 | 対象リスト空で警告のみ → ✅ 安全 |
| detail ボタンが table 外 | 空テーブルを表示 → ✅ 安全 |

`if(row){...}` ガード処理あり。null pointer エラーは発生しない。

### 5-3. td 内ネスト HTML（`<span>`, `<a>` nesting）

`td.textContent.trim()` で HTML タグを除去。

| TD内容 | 取得値 |
|--------|--------|
| `<span class="badge">有効</span>` | `有効` ✅ |
| `<a href="/sku/001">SKU-001</a>` | `SKU-001` ✅ |
| `<span style="color:red">エラー</span> <small>v2</small>` | `エラー v2` ✅ |
| `<strong>太字</strong> / <em>斜体</em>` | `太字 / 斜体` ✅ |

### 5-4. 「」パターンなし onclick の fallback

```javascript
var m = oc.match(/「(.+?)」モック動作/);
title = m ? m[1] : (btn.textContent || "").replace(/^[+\s]+/, "").trim().slice(0, 30) || "操作";
```

| ケース | 取得タイトル |
|--------|------------|
| 「」パターンなし・btnText あり | btnText を trim・30文字切捨て |
| 先頭空白 → trim後の文字列 | 正常取得 ✅ |
| 空 btnText | `操作` にフォールバック → generic ✅ |

## 6. バグリスト

### BUG-Q3-01 ⚠️ [medium] 「廃棄起票」が create モーダルに誤分類

| 項目 | 内容 |
|------|------|
| **ファイル** | `test2-mirror/pc/inventory/expiry_disposal.html` |
| **ボタン** | `廃棄起票` |
| **原因** | `classify()` が `起票`（create パターン）を `廃棄`（delete パターン）より先にヒット |
| **実際の動作** | create モーダル（コード / 名称 / 備考 の空フォーム + 「登録」ボタン） |
| **期待される動作** | delete モーダル（⚠️赤い警告 + 行データ + 「削除する」ボタン）または専用の廃棄確認モーダル |
| **影響** | 廃棄確認の UX が不自然。「登録」ボタンが廃棄操作に見える |
| **修正案** | classify() の create チェックを「廃棄起票」の複合ワード除外、または delete を create より前に評価 |

## 8. 修正提案サマリー

### 優先修正（medium）

```javascript
// 廃棄起票の誤分類修正案
function classify(title) {
  // delete チェックを create より前に移動
  if (/CSV|PDF|DL|ダウンロード|エクスポート|出力|印刷|発行/.test(title)) return "output";
  if (/詳細|プレビュー|明細/.test(title)) return "detail";  // ← 明細を追加
  if (/編集|変更|修正|更新/.test(title)) return "edit";
  if (/削除|失効|廃棄/.test(title)) return "delete";        // ← create より前に移動
  if (/新規|追加|登録|作成|起票/.test(title)) return "create";
  if (/テスト|監視|調査|確認|シミュレーション|再生/.test(title)) return "test";
  if (/実行|送信|承認|確定|開始|発動|連携|消込|統合|再送|招待|割当/.test(title)) return "exec";
  return "generic";
}
```

この変更による影響:
- `廃棄起票` → create から **delete** に変更 ✅
- `明細` ×20 → generic から **detail** に変更 ✅
- `緊急失効実行` → delete のまま（`失効` が `実行` より先）→ 要別途対応
