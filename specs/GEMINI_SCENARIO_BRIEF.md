## 0. このドキュメントについて

時吉さんが Gemini と壁打ちしながら、WMS「テストサイト2」用の **業務テストシナリオを数十件** 作成するための指示書。Gemini はまずこれを通読してから支援を始めること。

---

## 1. Gemini の役割

- WMS（倉庫管理システム）の業務テストシナリオ作成を、時吉さんと一緒に行う「**壁打ち相手 兼 ドラフト作成者**」。
- 時吉さんは物流のプロ（倉庫実務に精通・非エンジニア）。Gemini は、時吉さんが頭の中に持つ業務知識を、抜け漏れなく構造化されたシナリオに落とす手伝いをする。
- ゴール: 後述の「シナリオ・カタログ」に沿って業務貫通シナリオを数十件つくる。

---

## 2. プロジェクト背景

- **WMS** = 3PL対応（複数荷主受託型）の倉庫管理システム。倉庫事業者が複数荷主の在庫・入出庫・棚卸・請求を一元管理する。
- **テストサイト2** = WMS の全画面を、シナリオテストのために再現したプラットフォーム（PC / HT / スマホApp / 荷主ポータル）。
- **シナリオテストの目的** = 時吉さんが書いた業務シナリオを1年分のデータ込みで「ぶん回し」、**バグ ＋ 足りない機能を洗い出す**こと。
- だからシナリオは「正しく動くか」だけでなく「**この業務をやろうとしたら必要な画面・機能が無い**」も検出できる形で書く。「機能が無い」の発見も成果。

## 4. 「テストシナリオ」とは

1本のシナリオ = ある業務フローを最初から最後まで通す **手順（ステップ列）＋ 各所の期待結果**。

例:「マルキ食品の常温品を入荷予定登録 → HT端末で検品 → 格納 → 在庫が増えたことを確認」
- 各ステップ: 誰が・どの画面で・何をするか（業務の言葉で）
- 期待結果（アサーション）: そのステップ後、何がどうなっているべきか（在庫がN増える / ステータスが○○になる 等）

---

## 5. 良いシナリオの7原則

1. **業務語で書く** — 「IN-26060101」のような内部ID・技術用語を使わない。「マルキ食品の常温入荷を1件」のように業務の言葉で書く。（最終的な YAML 整形は2号が担当。時吉さんは業務語で書けばよい）
2. **各ステップに期待結果を必ず付ける** — 「〜する」で終わらせず「〜するとこうなるはず」まで書く。期待結果が無いステップはテストにならない。
3. **HT工程を厚く** — HT（ハンディ端末・キーエンス BT-A2000 4インチ / BT-A1000 3インチ）は本プロジェクトの根幹。HT工程のシナリオを最優先・多めに。実機でのバーコード実スキャン操作も想定する。
4. **1シナリオ = 1つの業務の筋** — 詰め込みすぎない。長い業務貫通は1本、個別機能の確認は1本。
5. **正常系だけでなく異常系・エッジも書く** — 在庫不足・品違い・期限切れ・容量超過 等。バグは異常系に出やすい。
6. **荷主分離（3PL）も検証する** — 「荷主Aでログイン中、荷主Bのデータが見えないこと」を確認するシナリオを入れる。
7. **「機能が無い」を恐れず理想の業務手順を書く** — テストサイト2 にその画面/機能が無ければ「不足機能」として洗い出される。それが目的。

## 6. シナリオ・カタログ（作成目標リスト）

このカタログ順に作成する。**G. HT工程を最優先・最多**に。

### A. 入荷系
- IN-1 通常入荷（ASN有）: 入荷予定登録 → HT検品 → 格納 → 在庫反映
- IN-2 ASN無し入荷: 予定外入荷の処理
- IN-3 入荷差異・数量過不足: 予定120 / 実検品110 の差異処理
- IN-4 入荷差異・品違い（wrong_item）
- IN-5 入荷差異・品質不良（外装つぶれ・期限切れ等）
- IN-6 ロット品入荷: 食品・賞味期限つき複数ロット
- IN-7 シリアル品入荷: 精密機器・個体管理
- IN-8 返品入庫（RMA）: 出荷済み品の返品受入
- IN-9 冷蔵品の入荷: 温度帯管理

### B. 在庫系
- ST-1 在庫照会（商品別 / ロット別 / ロケ別）
- ST-2 在庫移動（ロケ間）
- ST-3 棚卸・日々
- ST-4 棚卸・循環
- ST-5 棚卸・月次/期末（在庫凍結）
- ST-6 在庫調整（理由コード・荷主承認フロー）
- ST-7 補充指示（閾値ベース自動補充）
- ST-8 出荷止めロットの設定 → 引当除外確認

### C. 出荷系
- OUT-1 通常出荷 BtoB大口: 出荷指示 → 引当 → ウェーブ → HTピッキング → 梱包検品 → 積込 → 引渡（フル貫通）
- OUT-2 EC小口出荷: 多数の小口注文
- OUT-3 出荷の温度帯別自動分割
- OUT-4 ウェーブのエリア並行ピッキング
- OUT-5 ゾーンリレー方式ピッキング
- OUT-6 引当時の在庫不足（部分引当・バックオーダー）
- OUT-7 誤出荷の発覚と訂正
- OUT-8 出荷後返品
- OUT-9 出荷分割・梱包（個口数自動確定）
- OUT-10 用途区分（廃棄/サンプル/仕入先返品）の出荷
- OUT-11 FIFO引当の検証（古いロットから出る）

### D. 配送系
- DL-1 配送業者の自動振り分け（方面マスタ → 便候補抽出）
- DL-2 便容量超過時の自動振替
- DL-3 配送先指定業者の優先
- DL-4 配送追跡の取込

### E. 請求系
- BL-1 月次請求書生成・3期制（マルキ食品）
- BL-2 月次請求書生成・2期制（食品EC）
- BL-3 月次請求書生成・坪貸し（アパレル）
- BL-4 保管料の期分け計算検証
- BL-5 荷役料（入出庫件数ベース）
- BL-6 外部委託倉庫の委託支払計算
- BL-7 請求済み月の変更ブロック
- BL-8 請求書 PDF / CSV 出力

### F. 3PL・マルチテナント系
- TN-1 荷主切替（管理者が荷主A→B）
- TN-2 荷主分離（荷主Aユーザーに荷主Bのデータが見えない）★RLS
- TN-3 荷主別商品マスタ
- TN-4 共用ロケ vs 荷主専用ロケ
- TN-5 荷主ポータル — 在庫照会
- TN-6 荷主ポータル — 出荷状況 / 配送追跡
- TN-7 荷主ポータル — 請求書確認
- TN-8 荷主ポータル — 出荷指示登録

### G. HT工程（★根幹・最重点・最多に）
- HT-1 HT入荷検品（BT-A2000 4インチ）
- HT-2 HT入荷検品（BT-A1000 3インチ）= 小型機種での再現性
- HT-3 HT格納（ロケコードスキャン）
- HT-4 HTピッキング（スキャン式）
- HT-5 HT梱包・出荷検品
- HT-6 HT積込確認
- HT-7 HT引渡確認
- HT-8 HT棚卸
- HT-9 HT荷主切替
- HT-10 バーコードスキャン: JAN読取で商品特定
- HT-11 バーコードスキャン: ロケコード読取
- HT-12 バーコードスキャン: 伝票No読取
- HT-13 スキャン失敗・誤読時のエラー処理
- HT-14 HTでのロット現物表記入力
- HT-15 シリアル品のHTスキャン（精密機器）

### H. 異常系・エッジ
- EX-1 在庫ゼロでの引当
- EX-2 期限切れロットの引当除外
- EX-3 便容量超過
- EX-4 同一在庫への同時引当（競合）
- EX-5 不適合品ロケへの格納
- EX-6 月またぎ（期分け・繁忙期 7月/12月）の処理

## 7. シナリオの書き方テンプレート

時吉さんは下記の形で業務語で書けばよい（YAML への整形は2号が担当）。

```
■ シナリオ名: <わかりやすい名前>
- 対象荷主: <マルキ食品 / アパレル雑貨 / 食品EC / 精密機器>
- 区分: <PC / HT-4インチ / HT-3インチ / スマホApp / 荷主ポータル>
- 目的: <この業務で何を確認したいか>
- 前提: <開始時点のデータ状態を業務語で。例「マルキ食品の常温SKUが在庫にある」>
- 手順:
  1. <誰が> <どの画面で> <何をする> → <期待結果>
  2. ...
- 最終確認: <シナリオ完了後、業務ルール上こうなっているべき。
            例「在庫がN増加・入荷予定が完了ステータス・在庫履歴に入庫1件」>
- 異常系メモ: <この業務で起こりうる失敗があれば>
```

ポイント:
- 内部ID（IN-... / S-... 等）は書かない。業務語で。
- 期待結果は具体的に（数量・ステータス名まで）。
- HT のシナリオは、自動テストに加えて時吉さんが実機で操作するチェックリストにもなる。

## 補足: WMS の基本用語（Gemini が押さえておくべき最小限）

- **荷主**: 倉庫に在庫を預ける顧客企業。3PL = 1つの倉庫で複数荷主を扱う。
- **入荷 / 出荷**: 「受注」ではなく「出荷／出荷指示」と呼ぶ。
- **検品**: 入荷した現物の数量・品目を確認する作業。
- **格納（棚入れ）**: 検品済み品を保管ロケーションに収める作業。
- **引当**: 出荷指示に対して在庫を割り当てること。FIFO（先入先出）が基本。
- **ウェーブ**: 複数の出荷指示をまとめてピッキングする単位。
- **ロット**: 製造日・賞味期限などで区切られた在庫の単位（食品で重要）。
- **シリアル**: 個体ごとの識別番号（精密機器などで個体管理）。
- **期分け（3期制/2期制/坪貸し）**: 保管料の計算方式。荷主ごとに異なる。
- **HT**: ハンディターミナル（キーエンス BT-A2000 4" / BT-A1000 3"）。倉庫作業者が現場で使う。本システムの根幹。
