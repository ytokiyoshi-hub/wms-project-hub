## 1. ピッキングリスト (`/print/picking-list.html?id=<shipment_order_id>`)

### データソース

| API | テーブル（主） |
|-----|----------------|
| `GET /api/shipment-orders/:id` | `shipment_orders`, `picking_tasks` |

### ヘッダー部

| 印刷位置 | 項目名 | データキー | 備考 |
|----------|--------|-----------|------|
| タイトル | ピッキングリスト | — | 固定文字列 |
| 右上 | 伝票番号 | `order_no` | 例: SO-20260516-001 |
| 右上 | 発行日時 | システム日時 | 印刷時点の日時 |
| ヘッダー行 | 出荷予定日 | `scheduled_date` | YYYY-MM-DD |
| ヘッダー行 | 荷主 | `owner_code` + `owner_name` | 例: MK001 マルキ食品 |
| ヘッダー行 | 出荷先 | `customer_name` | 取引先マスタから |
| ヘッダー行 | 明細件数 / 合計数量 | `tasks.length` / `sum(tasks[].qty)` | 件数と合計ケース数 |

### 明細部

| 列 | 項目名 | データキー | 備考 |
|----|--------|-----------|------|
| 1 | 行番号 | 連番 | 1始まり |
| 2 | ロケ | `location_code` | ピッキング元ロケーション |
| 3 | 商品コード | `product_code` | 例: S-00321 |
| 4 | 商品名 | `product_name` | 商品マスタから |
| 5 | ロット | `lot_no` | ロット管理なしの場合は `-` |
| 6 | 数量 | `qty` | ケース単位 |
| 7 | 確認 | — | チェックボックス（手書き）|

### フッター部

| 項目名 | データキー | 備考 |
|--------|-----------|------|
| 備考 | `note` | 出荷指示のメモ |
| 担当者印 | — | 押印欄（固定） |
| 確認者印 | — | 押印欄（固定） |

## 5. 在庫明細書 N-10 (`/print/inventory-statement.html?owner=<owner_code>`)

> N-10: 荷主向けの月次在庫報告書。在庫数量を商品×ロケ×ロット単位で出力する。

### データソース

| API | テーブル（主） |
|-----|----------------|
| `GET /api/inventory[?owner=<code>]` | `stocks`, `skus`, `locations`, `lots` |

### ヘッダー部

| 印刷位置 | 項目名 | データキー | 備考 |
|----------|--------|-----------|------|
| タイトル | 在庫明細書 | — | 固定文字列 |
| 右上 | 発行番号 | `INV-STMT-${YYYYMM}` | 例: INV-STMT-202605 |
| 右上 | 発行日時 | システム日時 | |
| ヘッダー行 | 基準日時 | システム日時 | データ取得時点 |
| ヘッダー行 | 対象荷主 | `owner_code` または `全荷主` | フィルタなし時は全荷主 |
| ヘッダー行 | 行数 | `inv.length` | |
| ヘッダー行 | 合計数量 | `sum(inv[].qty)` | ケース単位 |

### 明細部

| 列 | 項目名 | データキー | ソート順 |
|----|--------|-----------|----------|
| 1 | 行番号 | 連番 | — |
| 2 | 商品コード | `product_code` | 第1ソートキー |
| 3 | 商品名 | `product_name` | |
| 4 | 荷主 | `owner_code` | |
| 5 | 温度帯 | `temperature_zone` | 常温/冷蔵/冷凍 |
| 6 | ロケ | `location_code` | 第2ソートキー |
| 7 | ロット | `lot_no` | ロット管理なしは `-` |
| 8 | 数量 | `qty` | ケース単位 |

### フッター集計部

| 項目名 | データキー | 備考 |
|--------|-----------|------|
| 合計 | `sum(inv[].qty)` | 全行合計数量 |
| 注記 | — | `本明細書は {発行日時} 時点のシステム在庫を出力したものです` |

## 補足: 帳票共通事項

| 項目 | 内容 |
|------|------|
| 用紙サイズ | A4 縦 |
| 出力形式 | ブラウザ印刷（`window.print()`）→ PDF 保存可能 |
| 文字コード | UTF-8 |
| フォント | Noto Sans JP（印刷用）、Roboto Mono（コード類） |
| 帳票 CSS | `/css/print.css` |
| XSS 対策 | `WMS.escape()` で全出力値をエスケープ |

### Phase 9 PDF 出力実装時の追加検討事項

- `print.css` の `@media print` で `no-print` クラスを非表示にする現行方式を維持
- PDF 保存は `window.print()` + ブラウザのPDF保存機能で対応（Phase 9 中盤以降）
- 帳票番号の採番ルール（連番 vs 日付ベース）は時吉さんの判断を仰ぐこと
- 請求書の振込先・お支払期限は倉庫マスタから取得できるよう Phase 9 で仕様化要
