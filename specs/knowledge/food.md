# 食品 3PL 業界 物流ナレッジ

> Phase 9-N1 収集。食品物流特有の要件（温度帯・賞味期限・トレーサビリティ・HACCP）を WMS 設計に反映するための参照資料。

---

## 1. 温度帯管理

### 1-1. 3温度帯（業界標準）

| 温度帯 | 呼称 | 温度基準 | 代表的な対象品目 |
|-------|------|---------|--------------|
| 常温（ドライ） | ドライ / 常温 | 10℃〜20℃ | 米・缶詰・調味料・チョコレート菓子・マヨネーズ |
| 冷蔵（チルド） | チルド | -5℃〜5℃ | 乳製品・精肉・固形油脂（マーガリン等）・生鮮品 |
| 冷凍（フローズン） | フローズン | -15℃以下 | 冷凍食品・パン生地・アイスクリーム（-18℃以下）・冷凍マグロ（-55℃以下） |

> 冷凍食品には品目ごとの細分類あり：C1級（-10℃以下）/ F1級（-18℃以下）/ F3級（-55℃以下）

### 1-2. 設備要件

- 各温度帯に対応した専用保管スペース（冷却設備・空調・断熱構造）
- ドッグシェルター（密閉型入出庫口装置）による温度漏れ防止
- 温度を保つ扉・什器（冷凍倉庫専用フォークリフト等）
- 温度ロガーによる**継続的な温度記録**（HACCP 要件）

### 1-3. WMS での管理要件

- 商品マスタに**温度帯区分**を持つ（例: `temperature_zone: enum('ambient', 'chilled', 'frozen')`）
- ロケーションマスタに温度帯を紐付け → ロケーション割当時の**温度帯バリデーション**
- 入出庫指示で温度帯を表示 → ピッカーが誤ロケーションに入れないようアラート

## 2. 賞味期限・消費期限管理

### 2-1. FEFO（First-Expire, First-Out）

食品物流における出庫原則。**期限が最も近いロットを優先出庫**する。

- 一般物流の FIFO（先入れ先出し）とは異なる
- WMS はロット単位で期限を管理し、出庫指示を期限の古い順で自動生成
- OCR 機能による賞味期限の自動読取（ハンディ端末カメラ）でヒューマンエラーを削減

### 2-2. 3分の1ルール（日本独自の商習慣）

```
製造日 ─────────────────── 賞味期限
        ↑              ↑
    納品期限         販売期限
 （製造日+1/3期間）（製造日+2/3期間）
```

- **第1段階**（製造日〜1/3）：卸業者が小売店に納品しなければならない「納品期限」
- **第2段階**（1/3〜2/3）：小売店が店頭に陳列できる「販売期限」
- **第3段階**（2/3〜期限）：消費者が賞味できる期間

**WMS 実装ポイント：**

- 出荷時に「残日数チェック」を実施 → 残日数が設定閾値を下回る場合は出荷不可
- 取引先ごとに残日数閾値を設定可能（例：A社は60日以上残存を要求）
- 閾値を超えた在庫は自動的に「出荷停止ステータス」へ移行 → 廃棄処理フロー

### 2-3. 期限切れ在庫アラート

| アラートレベル | 残日数 | 対応 |
|------------|------|------|
| 警告（yellow） | 閾値の150%以内 | 出荷促進アクション通知 |
| 危険（red） | 閾値以内 | 出荷停止・廃棄申請 |
| 廃棄済（expired） | 0以下 | 在庫から除外・廃棄記録 |

## 3. トレーサビリティ

### 3-1. 日本の法的枠組み

| 法律 | 対象 | 主な要件 |
|-----|------|---------|
| **牛トレーサビリティ法** | 牛肉 | 10桁の個体識別番号。飼育〜流通〜消費まで追跡義務 |
| **米トレーサビリティ法** | 米・米加工品 | 入出荷記録・産地記録の保存（伝票等で対応可） |
| **食品衛生法** | 全食品事業者 | 仕入先・販売先の記録保存（基礎トレーサビリティ・努力義務） |

### 3-2. 基礎トレーサビリティの要件

1. **入荷記録**：入荷日・仕入先・品目・ロット番号・数量
2. **出荷記録**：出荷日・出荷先・品目・ロット番号・数量
3. **ロット識別**：ロット番号の表示（バーコード等）
4. **ロット対応**：入荷ロットと製造ロットの紐付け
5. **記録保存期間**：品目・法律により異なる（例：米は3年）

### 3-3. リコール対応要件

- 問題ロットの**上流（仕入先）**・**下流（出荷先）**の双方向追跡
- 「このロットの在庫がどの出荷先に何個出たか」を即座に特定できること
- WMS でロット単位の在庫・移動履歴を完全保持（削除禁止）

### 3-4. WMS 実装ポイント

- `lot_number` を全入出庫トランザクションに必須フィールド化
- ロット単位の在庫引当（ロット指定ピッキング）
- 追跡レポート：ロット → 出荷先一覧、出荷先 → ロット一覧（双方向）
- QR / バーコードスキャンによるロット読取を入庫・検品・出庫の全工程で実施

## 4. HACCP（ハザード分析重要管理点）

### 4-1. 義務化の概要

- 2018年：食品衛生法改正により HACCP 義務化決定
- **2021年6月：全食品等事業者への HACCP 義務化完全施行**
- 対象：食品の保管・運搬を行う 3PL 事業者も含む

### 4-2. 物流・倉庫業における適用

物流企業の HACCP は「**一般衛生管理計画**」を中心とした対応。

| 管理項目 | 具体的な要件 |
|--------|-----------|
| **温度管理** | 冷蔵・冷凍倉庫の温度を常時一定に保ち、定期的に記録 |
| **衛生管理** | 倉庫・車両の清掃・消毒記録の作成と保存 |
| **異物管理** | 虫・ネズミ等の侵入防止設備と定期的な防虫・防鼠記録 |
| **交差汚染防止** | 温度帯・アレルゲン別の保管エリア分離 |
| **記録保存** | 衛生管理計画・実施記録の定期的な検証と見直し |

### 4-3. WMS での記録支援

- 温度ロガーデータをシステムに自動取込（設備連携 API）
- 異常温度発生時のアラート・作業停止フラグ
- 清掃・消毒チェックリストのデジタル化と日付スタンプ付き保存
- 衛生管理記録のエクスポート機能（保健所への報告用）

## 5. アレルゲン管理

### 5-1. 特定原材料 8品目（表示義務）

えび・かに・くるみ・小麦・そば・卵・乳・落花生

### 5-2. 倉庫での管理要件

- アレルゲン含有商品の**専用保管エリア**または明確な区画分離
- ピッキング時のアレルゲン混載アラート
- 出荷検品時のアレルゲン確認チェック

### 5-3. WMS 実装ポイント

- 商品マスタにアレルゲン区分を登録（8品目フラグ）
- 注文に特定アレルゲン含有商品が混在する場合は警告表示
- 代替品出荷時のアレルゲン差異チェック（別アレルゲンに切り替わる場合は要確認）

## 6. ロット管理（まとめ）

### WMS に必要なロット管理機能

| 機能 | 内容 |
|-----|------|
| ロット入庫登録 | 入庫時にロット番号・製造日・賞味期限を登録 |
| ロット別在庫照会 | ロット単位での在庫数量・保管ロケーション確認 |
| ロット引当 | 出荷指示時に FEFO 順でロットを自動引当 |
| ロット指定出荷 | 取引先指定ロットを優先出荷（特注対応） |
| ロット追跡レポート | 上流（仕入）・下流（出荷）の双方向追跡 |
| 期限アラート | 期限切れ予告・出荷停止の自動化 |
| リコール対応 | 問題ロットの全出荷先を即時リスト出力 |

## 7. 法的・資格要件サマリー

| 要件 | 根拠法 | 対象 | 対応方法 |
|-----|-------|------|---------|
| 冷蔵倉庫業の許可 | 食品衛生法（食品の冷凍又は冷蔵業） | 冷蔵・冷凍倉庫事業者 | 保健所への営業許可申請 |
| 倉庫業登録 | 倉庫業法 | 営業倉庫事業者 | 国土交通省へ登録申請・倉庫管理主任者選任 |
| HACCP 義務 | 食品衛生法（2021年〜） | 全食品等事業者 | 一般衛生管理計画策定・記録保存 |
| トレーサビリティ | 牛トレ法・米トレ法・食品衛生法 | 品目に応じた事業者 | 入出荷記録・ロット番号の管理と保存 |
| アレルゲン表示 | 食品表示法 | 食品製造・販売事業者 | 保管・出荷時の交差汚染防止管理 |

## 8. WMS 設計への適用優先度

| 要件 | 優先度 | 理由 |
|-----|--------|------|
| 温度帯ロケーション管理 | **高** | 3温度帯の混在管理は基本機能 |
| FEFO 出庫指示 | **高** | 食品では FIFO より重要 |
| 賞味期限・ロット登録 | **高** | トレーサビリティの根幹 |
| 3分の1ルール チェック | **高** | 日本市場特有・未対応は商習慣違反 |
| HACCP 温度記録連携 | **中** | 義務化済みだが手動記録でも代替可 |
| アレルゲンアラート | **中** | 混載リスクが高い品目取扱時に必須 |
| リコール追跡レポート | **中** | 問題発生時の対応速度に直結 |
| 廃棄フロー自動化 | **低** | 期限切れ在庫処理の効率化（後回し可） |
