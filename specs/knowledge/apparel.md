# アパレル 3PL 業界 物流ナレッジ

> Phase 9-N2 収集。アパレル物流特有の要件（季節商品・SKU数膨大・タグ付け・返品・VAS）を WMS 設計に反映するための参照資料。

---

## 1. 季節商品管理

### 1-1. シーズン構造（業界標準）

| シーズン | 略称 | 期間（日本市場） | 代表品目 |
|--------|------|--------------|--------|
| Spring/Summer | SS | 2月〜8月 | 薄手トップス・ショートパンツ・水着 |
| Fall/Winter | FW | 8月〜2月 | コート・セーター・ブーツ |
| Resort / Cruise | RST | 11月〜1月 | 旅行・リゾート向け軽アウター |
| Pre-season（先行） | PRE | 各本番シーズンの1〜2ヶ月前 | 別注・先行予約商品 |

> シーズンをまたぐ在庫（キャリーオーバー品・定番品）は通年扱い。

### 1-2. 季節在庫の特性

- **入荷集中**：シーズン初期（2月・8月）に大量一括入荷 → 入荷バースとマンパワーの計画が重要
- **出荷集中**：シーズン中盤〜後半にかけてピーク → 在庫回転率が急速に上昇
- **シーズンエンド在庫**：シーズン末の残在庫はセール・アウトレット・返品・廃棄に振り分け
- **空きスペース問題**：シーズン切替期に一時的にロケーション稼働率が極端に落ちる

### 1-3. WMS での管理要件

- 商品マスタに `season` フィールド（SS / FW / RST / PRE / PERENNIAL）
- シーズン別在庫サマリーレポート
- シーズンエンド処理：残在庫の一括ステータス変更（SALE / RETURN / DISPOSE）
- シーズン切替時の保管ロケーション移動指示（スペース最適化）

## 2. SKU 数・品番管理

### 2-1. アパレル SKU の爆発的拡張

1つの「商品」は **品番 × カラー × サイズ** の組み合わせで SKU が生まれる。

```
例：ダウンジャケット 1型
  × カラー 5色（BLK / NVY / RED / WHT / GRY）
  × サイズ 5サイズ（XS / S / M / L / XL）
  = 25 SKU

ブランドが200型保有 → シーズン在庫SKU = 5,000〜
```

- **中規模アパレルブランド**：SKU数 3,000〜20,000
- **大規模 3PL（複数ブランド）**：SKU数 100,000 超は珍しくない

### 2-2. 品番体系（業界標準）

| コード種別 | 桁数 | 内容 |
|---------|------|------|
| 品番（スタイルナンバー） | 6〜12桁 | デザイン・型を識別。シーズン・品種コードを含むことが多い |
| JAN コード | 13桁 | JANコード（バーコード）。カラー・サイズまで識別 |
| UPC（海外ブランド） | 12桁 | 米国規格。輸入ブランドは UPC → JAN 変換が必要 |
| 社内 SKU コード | 任意 | 3PL 側が付番する場合あり（ブランドコード接頭辞付きなど） |

### 2-3. カラー・サイズのバリエーション管理

| 属性 | 業界の複雑性 | WMS 実装ポイント |
|-----|-----------|--------------|
| カラー | カラー名と実際の色が一致しない（例：「オフホワイト」「ミルクホワイト」が別 SKU）| カラーコード必須。人間が識別しやすいカラー名も保持 |
| サイズ | 数字（34/36/38）・英字（XS/S/M/L）・日本語（小/中/大）が混在 | ブランド別サイズテーブル + ソート用サイズ順序（size_order）フィールド |
| シーズン × 品番 | 毎シーズン型番が変わる（似て非なる品番が大量発生） | 前シーズン品番との紐付け（`parent_style_id`）が在庫照会・継続管理に有用 |

## 3. タグ付け・ラベリング（VAS）

### 3-1. タグの種類

| タグ種別 | 目的 | 取付方法 |
|--------|------|--------|
| 下げ札（ハンガータグ） | 品番・価格・JAN 表示 | タグガン（プラスティック留め具）またはリボン/ひも |
| 品質表示タグ（洗濯ラベル） | 素材・洗濯方法（法律義務） | 縫い付け（メーカー工場が対応） |
| 値札シール | 価格・バーコード | シールで貼り付け |
| セキュリティタグ | 万引き防止（EAS タグ） | タグガン・ループ・ラチェット等 |
| RFID タグ | 在庫可視化・ゲート一括読取 | ラベル型貼付またはタグへの埋め込み |

### 3-2. タグ付け作業の WMS 管理要件

```
入荷 → 検品 → タグ付け（VAS）→ ハンギング or 折り畳み → 保管
```

- タグ付け工数は商品ごとに異なる（単品タグ vs 上下セットのタグ付け）
- **タグ付け指示書**：荷主が指定するタグ仕様（場所・方法・レイアウト）を WMS で管理
- 作業実績記録：タグ付け完了数・担当者・時間 → 荷主への請求根拠
- **タグ変更**：セール時に値札シール貼り替え（割引価格への変更）も VAS として受注

### 3-3. 品質表示（法的義務）

- **家庭用品品質表示法**：繊維製品は品質表示義務あり
- 洗濯ラベルの国際記号（ISO 3758）に準拠
- 3PL は輸入時に表示確認の責任範囲を荷主と契約で明確化しておく必要がある

## 4. ハンギング保管 vs 折り畳み保管

### 4-1. 保管形態の違い

| 形態 | 対象品 | 設備 | 面積効率 |
|-----|-------|------|--------|
| ハンギング（吊り保管） | コート・スーツ・ドレス・デリケート素材 | ハンガーラック・スライダーレール | 低い（縦方向に場所を取る） |
| 折り畳み（フラット保管） | Tシャツ・ニット・デニム・下着・靴下 | 通常棚・段ボール | 高い |
| 箱保管 | 靴・バッグ・アクセサリー | 棚・パレット | 中程度 |

### 4-2. WMS での管理要件

- ロケーションマスタに**保管形態**属性（hanging / folded / boxed）
- 商品マスタに**推奨保管形態**を登録
- ハンギング商品の入荷時：ハンガー本数のカウント + ロケーション空きスペース管理
- ハンギング保管では在庫単位が「1点 = 1ハンガー」で管理

## 5. 返品・リターン処理

### 5-1. 返品率の高さ

アパレルはEC返品率が特に高い業界（日本：EC返品率15〜25%が目安、欧米は30〜40%超）。

### 5-2. 返品フロー

```
顧客返送 → 返品受付 → 検品（汚れ・傷・タグ有無） → ランク判定 → 処置
```

| ランク | 状態 | 処置 |
|------|------|------|
| A品（新品同様） | 未使用・タグあり | 再入庫・再販 |
| B品（若干使用感） | 試着程度・タグなし | アウトレット出荷 / 一時保留 |
| C品（使用・汚損あり） | 洗濯・修繕要 | 修繕後再販 / 廃棄申請 |
| 不良（破損・異品） | 明らかな異常 | 荷主へ返還または廃棄 |

### 5-3. WMS での返品管理要件

- 返品受付伝票（RMA番号）による追跡
- 返品検品結果の記録（ランク・理由コード・写真）
- ランク A品の自動再入庫処理（在庫復元）
- 返品在庫の保管ロケーション（検品待ちエリア・アウトレットエリア）の分離
- 返品集計レポート（荷主別・SKU別・理由コード別）

## 6. EC 物流との連携

### 6-1. EC 特有の出荷要件

| 要件 | 内容 |
|-----|------|
| 小口多頻度出荷 | 1注文=1〜数点。BtoC 向けに1日数百〜数千件の小口出荷 |
| 梱包指示 | 荷主ブランドの指定梱包材・ノベルティ同梱・メッセージカード封入 |
| 翌日配送対応 | 午後○時までの注文を当日出荷する締め切り管理 |
| ギフト包装 | のし・ギフトボックス・リボン対応（有料 VAS） |
| 出荷通知 | 配送伝票番号をEC プラットフォームへ自動連携 |

### 6-2. EC プラットフォーム連携

| プラットフォーム | API 連携ポイント |
|--------------|--------------|
| Shopify | 注文取込・在庫同期・出荷通知 |
| ZOZOTOWN | 専用 EDI / FTP 連携 |
| Amazon (FBA/FBM) | SP-API 経由 |
| 楽天市場 | RMS API |
| 自社 EC | REST API |

> **在庫過多・二重受注リスク**：複数 EC サイトで同一在庫を共有する場合、リアルタイムの在庫同期が必須。

## 7. RFID の活用（アパレル業界の主要トレンド）

### 7-1. アパレル RFID の特性

- 繊維製品は電波を吸収しにくい素材が多く、食品（水分多い）より RFID 認識率が高い
- RFID タグのコスト：1枚 15〜30円まで下落（2024年時点）
- ZARA・H&M・UNIQLO 等のファストファッションで大規模導入済

### 7-2. RFID 活用ポイント

| 活用場面 | 効果 |
|--------|------|
| 入荷検品 | ゲート通過だけで数百点を数秒で一括計数（照合精度 99%以上） |
| 棚卸 | ハンドヘルド RFID リーダーで倉庫棚卸を従来比 1/10 の時間で完了 |
| ピッキング確認 | 誤ピック防止（想定外 SKU が混入するとアラート） |
| EAS（万引き防止） | レジ通過時に自動 EAS 解除（タグ兼用） |
| 店舗在庫管理 | 倉庫と店舗の在庫を統合管理（Omni-channel 対応） |

### 7-3. WMS 実装ポイント

- 商品マスタに `rfid_enabled` フラグ
- RFID タグ番号（EPC）を入庫時にロケーション・SKU に紐付け
- ゲートリーダーからの受信データを自動処理するインターフェース
- RFID 非対応品は従来バーコードでカバーする混在設計

## 8. 在庫管理の特殊事情

### 8-1. 単品管理 vs ロット管理

| 管理方式 | 対象 | 特徴 |
|--------|------|------|
| 単品管理（シリアル管理） | 高級品・バッグ・ジュエリー | 1点ごとにシリアルナンバーを持つ。盗難・偽造防止目的 |
| ロット管理 | 消耗品・大量品 | バッチ単位で管理 |
| SKU 管理（一般） | 一般アパレル | SKU 単位での数量管理（単品追跡不要） |

### 8-2. 在庫差異リスク（アパレル特有）

| リスク要因 | 内容 | 対策 |
|---------|------|------|
| サイズ・カラー誤ピック | 似た商品の取り違え | JAN バーコードスキャン確認の徹底 |
| タグの剥がれ | ハンギング保管中に下げ札が脱落 → 商品が不明品化 | タグガン不使用時はひも結び必須 |
| 季節末の棚卸精度 | シーズンエンドセールで出荷量急増 → 棚卸タイミングがずれる | セール前後に棚卸を必ず実施 |
| 返品の在庫復元ミス | 返品品を未検品のまま在庫に戻す | 返品検品完了フラグが立つまで在庫復元禁止 |

### 8-3. 特殊素材・保管条件

| 素材・品目 | 保管上の注意 |
|---------|-----------|
| ニット | 重ね折り禁止・ハンギングは型崩れするため折り畳み推奨 |
| レザー（革製品） | 高温多湿を避ける・通気確保 |
| ダウン・羽毛製品 | 圧縮保管禁止・専用不織布カバー |
| シルク・デリケート素材 | 直接照明・蛍光灯による色変防止 |
| 毛皮 | 防虫・防湿・低温保管 |

## 9. 流通 BMS・EDI 対応

### 9-1. 流通 BMS の概要

| 帳票種別 | 内容 |
|--------|------|
| 発注（210） | 小売→卸・3PL への発注 |
| 出荷指示（220） | 荷主→3PL への出荷指示 |
| 出荷（856） | 3PL→小売への出荷通知（ASN） |
| 請求（810） | 卸・3PL→小売への請求 |
| 受領（880） | 小売が受領を確認 |

### 9-2. アパレル業界の EDI 特徴

- **アパレル業界標準 EDI**：流通 BMS（XML 形式）が主流
- 百貨店・SC（ショッピングセンター）は独自 EDI を要求するケースあり
- JAN コードが共通識別子として機能（ブランド内品番では通じない）

## 10. 法的・資格要件サマリー

| 要件 | 根拠 | 対象 | 対応方法 |
|-----|-----|------|--------|
| 品質表示義務 | 家庭用品品質表示法 | 繊維製品の販売者・製造者 | 洗濯ラベル・品質表示の確認。3PL は荷主から受領した商品の表示確認を行う |
| 倉庫業登録 | 倉庫業法 | 営業倉庫事業者 | 国土交通省への登録・倉庫管理主任者選任 |
| 古物商許可 | 古物営業法 | 中古衣類を取り扱う場合 | 警察署への許可申請 |
| 化粧品・雑貨の薬事 | 薬機法 | 化粧品・機能性インナー等 | 取り扱い可能品目の確認・要事前確認 |
| 輸入貨物取扱 | 関税法・外為法 | 輸入アパレル | 通関業者との連携・原産国管理 |

## 11. WMS 設計への適用優先度

| 要件 | 優先度 | 理由 |
|-----|--------|------|
| カラー×サイズ SKU 管理 | **高** | アパレルの基本。SKU爆発に対応できないと全機能が機能しない |
| JAN バーコードスキャン | **高** | 誤ピック防止の最終防衛ライン |
| シーズン管理 | **高** | シーズン切替を WMS が把握していないと入出荷指示がおかしくなる |
| 返品検品フロー | **高** | EC 返品率が高くEC事業では必須 |
| タグ付け VAS 管理 | **高** | 3PL の付加価値サービスとして収益化・工数管理に必須 |
| ハンギング保管対応 | **中** | ハンギング品を扱う倉庫では必須。折り畳みのみなら後回し可 |
| EC プラットフォーム在庫同期 | **中** | EC 荷主を受託するなら必須。B to B 専業なら不要 |
| RFID 対応 | **中** | 大手ブランド・ZARAクラスでは必須になりつつあるが、中小荷主はまだバーコード主流 |
| 流通 BMS / EDI 連携 | **中** | 百貨店・SC 荷主を受託する場合は必須 |
| 単品シリアル管理 | **低** | 高級ブランドのみ。通常アパレルには不要 |
