## QA確定事項（3号ヒアリング・QA-2②）

### 入庫カウントの基準（確定）

**入庫カウントは「検品完了時点」を基準とする。**

| 項目 | 確定内容 |
|------|---------|
| カウント基準 | **検品完了時点**（棚入れ実行タイミングではない） |
| 根拠 | プロトの B案（棚入れ完了基準）を採用しないことを QA-2② で確定 |
| 影響 | `inventory` への INSERT タイミングは棚入れ実行時だが、「入庫済み」としての業務カウントは検品完了時点で計上 |

> ⚠️ 注意：プロトタイプ段階では「棚入れ完了」を基準とする B案を前提としていた。実装時はこの変更点を踏まえ、検品完了時点での `inbound_movements.status = 'inspected'` 更新 → 入庫カウント計上 のフローを実装すること。

#### 影響するテーブル・カラム

```sql
-- inbound_movements: 検品完了ステータス追加
-- status: 'arrived' → 'inspecting' → 'inspected'（← 入庫カウント計上点） → 'putaway_pending' → 'putaway_done'
-- 検品完了時点で inventory_count_at = NOW() を記録する
ALTER TABLE inbound_movements
  ADD COLUMN inventory_count_at TIMESTAMPTZ;  -- 入庫カウント計上日時（検品完了時点）
```
