### QA-14: 配送便容量超過時の振替処理（確定）

配送便の積載容量を超過した場合、**自動振替候補を表示し、最終確定は人が行う**。

#### フロー

```
配送便容量チェック（割当時）
  ↓
超過検出（allocated_volume > carrier_capacity）
  ↓
システムが代替便の候補を自動算出・表示
  ↓
担当者が候補一覧から振替先を選択
  ↓
担当者が確定操作 → 振替完了
```

#### 振替候補の自動算出ロジック

| 条件 | 優先順位 |
|------|---------|
| 同一方面・同一出荷日 | 1位 |
| 同一方面・翌営業日 | 2位 |
| 別方面（エリア近接） | 3位（参考提示のみ） |

- 容量に空きがある便のみ候補表示
- 最終確定は必ず人が行う（自動確定禁止）
- 振替後は出荷指示明細の `carrier_id` / `delivery_date` を更新

### QA-15: カレンダー連動の配送振分（確定・B変形）

カレンダー連動の配送振分は**B変形（配送可能エリア検証付きの提案方式）**として実装する。

#### B変形の定義

「B変形」＝カレンダーの配送可能日・可能エリアを考慮した上で**振分案を提案**し、担当者が確定する方式。

#### フロー

```
出荷指示登録（ship_date 指定）
  ↓
配送カレンダー参照
  carrier_calendar テーブル: (carrier_id, date, available, available_regions[])
  ↓
ship_date に配送可能かチェック
  ↓
配送可能 → 通常振分
配送不可（休配日 / 配送不可エリア） → 次の配送可能日を自動検索
  ↓
代替日・代替便の提案リストを生成
  ↓
担当者が提案から選択・確定
```

#### 配送カレンダーテーブル

| カラム | 型 | 説明 |
|--------|-----|------|
| id | BIGSERIAL | PK |
| carrier_id | BIGINT | FK→carriers |
| date | DATE | 対象日 |
| available | BOOLEAN | 配送可否（false=休配日） |
| available_regions | TEXT[] | 配送可能エリアコード一覧（NULL=全エリア可） |
| note | TEXT | 備考（例：台風・年末年始） |

- カレンダーは年1回一括登録（運送会社から提供される休配日リスト）
- 臨時休配日は随時追加可能
- 配送可能エリア検証：`region_code = ANY(available_regions)` で判定
