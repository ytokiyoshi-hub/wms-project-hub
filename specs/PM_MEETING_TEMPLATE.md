# 使い方

## 1. 前日夕方〜夜の流れ

1. まーちゃんが当日の振り返り資料を自動生成（ナイト・ミーティング前段）
2. 時吉さん×まーちゃんが画面見ながら順に確認
3. テンプレに沿って翌日プラン確定
4. 確定内容を kurokun_memo に decision/critical で記録
5. 翌日プランの pending タスクに tag を付ける（例：tag: tomorrow-priority）

## 2. 翌朝の自動展開

1. 各 runner が起動時に直近の PM ミーティング memo を読む
2. 「翌日プラン」セクションのタスクを自分のキューから消化
3. PM ミーティング時刻になったら裏で稼働継続（異常感知最適）

## 3. 「漏らすな」を仕組み化

時吉さん哲学：超重要ミーティングは絶対に漏らさない。

実装：
- テンプレの全項目を埋める（空欄は「なし」と明記）
- 終わったら kurokun_memo にコピー（decision/critical）
- specs/PM_MEETING_TEMPLATE.md は最新版を維持
- 過去議事録は `meetings/` ディレクトリに日付別保存

# 「裏で稼働継続」の運用

時吉さんの指摘で重要：**ミーティング中こそ AI 並列稼働させる**（異常感知最適）。

```
PM ミーティング中の運用：
  ├─ 副社長まーちゃん：時吉さんと対話（このセッション）
  ├─ こーちゃん（PM）：通常稼働継続・自分のキュー消化
  ├─ さーちゃん（DB）：通常稼働継続
  ├─ にーちゃん（QA）：通常稼働継続
  └─ ナイト・ミーティング自動 cron は時間帯まで休止
```
