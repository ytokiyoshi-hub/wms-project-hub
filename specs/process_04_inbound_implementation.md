## スコープ

**入荷から在庫加算までの一連の業務フロー**を実装する工程。

業務シナリオ：
1. ASN 受信（または現物到着）→ 入荷予定登録
2. 現物到着 → 検品（荷主の検品方式に従う）
3. 検品結果反映 → 棚入れ指示生成
4. 棚入れ実施 → 在庫加算

## QA確定事項（3号ヒアリング・QA-7）

### ロット番号設計（確定）

| 項目 | 確定内容 |
|------|---------|
| ロット番号の基準 | **現物表記準拠**（荷主が現物に表記するロット番号をそのまま入力） |
| 倉庫自動採番 | **廃止**（RT-プレフィックス等は使用しない） |
| 多重ロット対応 | `lot_1`, `lot_2`, `lot_3`... の複数欄を持つ（使用欄数は荷主設定） |
| ロット欄名称 | 荷主ごとに変更可能（例: 「製造ロット」「入荷日」「ベンダーコード」等） |
| ロット番号書式 | **英数字のみ**（記号・全角・スペース不可） |
| ロット必須制御 | 荷主マスタの `lots_required` フラグ（BOOLEAN）で強制。`true` の場合、ロット未入力は登録不可 |

#### 影響テーブル（拡張内容）

```sql
-- lots テーブル：多重ロット欄対応
ALTER TABLE lots
  ADD COLUMN lot_1 TEXT,          -- 第1ロット欄（現物表記準拠・英数字のみ）
  ADD COLUMN lot_2 TEXT,          -- 第2ロット欄（荷主設定で活性化）
  ADD COLUMN lot_3 TEXT;          -- 第3ロット欄（荷主設定で活性化）
-- CHECK制約: lot_1 ~ lot_3 は英数字のみ（^[A-Za-z0-9]+$）

-- owners テーブル：ロット制御フラグ追加
ALTER TABLE owners
  ADD COLUMN lots_required    BOOLEAN DEFAULT false,   -- ロット必須フラグ
  ADD COLUMN max_lot_columns  INT DEFAULT 1,            -- 使用するロット欄数（1〜3）
  ADD COLUMN lot_1_label      TEXT DEFAULT 'ロット番号', -- 第1欄の表示名
  ADD COLUMN lot_2_label      TEXT,                     -- 第2欄の表示名
  ADD COLUMN lot_3_label      TEXT;                     -- 第3欄の表示名
```

> 旧 `lot_number` カラムが存在する場合は `lot_1` へのマイグレーションが必要。
> B-7 ユーザー・ロールマスタ確定後に権限周りとの連携を整備。

## 実装順序の推奨

```
F-401（入荷予定登録）
  ↓
F-402（入荷登録）
  ↓
F-403（検品）  ←─ F-404（差異処理・並行）
  ↓
F-405（棚入れ指示）
  ↓
F-406（棚入れ実行＋在庫加算）

F-407（入荷返品）：F-403 完了後にいつでも着手可
```
