### 5. 改善提案

#### 発見された問題（優先度順）

---

**【優先度：高】A. 最終更新日が古い**

- **場所：** index.html 541行目
- **現状：** `最終更新：2026/05/02`
- **正しい値：** `最終更新：2026/05/04`（最終コミット日）
- **影響：** サイト閲覧者が最新情報を誤認する可能性あり
- **修正方法：** index.html の `last-updated` div のテキストを `2026/05/04` に変更

```html
<!-- 修正前 -->
<div class="last-updated">最終更新：2026/05/02</div>

<!-- 修正後 -->
<div class="last-updated">最終更新：2026/05/04</div>
```

---

**【優先度：高】B. CLAUDE.md と index.html の工程1状態が不一致**

- **CLAUDE.md（85行目）：** `工程1（マスタ管理）：✅ 完了（10/10マスタ）`
- **index.html（proc-card）：** 工程1 = 進行中（2/10, 20%）
- **KPI値：** 2/16（工程12・14のみを完了としてカウント）
- **実態確認が必要：** 工程1のマスタ管理レビューは本当に完了しているか？
  - 完了であれば：index.htmlの工程1を「完了」に変更、KPIを3/16に修正
  - 未完了であれば：CLAUDE.mdの85行目を「進行中（2/10）」に修正

---

**【優先度：中】C. フェーズ2説明文の誤り**

- **場所：** index.html 588行目
- **現状：** `課金・請求 完了、その他14工程未着手`
- **問題：** 工程14（その他出庫）も完了しているため、「未着手」は13工程が正しい
- **修正案：** `課金・請求・その他出庫 完了、その他13工程未着手`

---

**【優先度：低】D. 画面モック内訳テキストの更新漏れ**

- **場所：** index.html 570行目
- **現状：** `PC 53 / HT 5 / Mobile 3 / Portal 9`（合計：70）
- **問題：** KPI合計値は75なのに内訳の合計が70で5画面分の差がある
- **原因：** 2026-05-04 に10画面追加した際、合計値（65→75）は更新されたが、内訳テキストが更新されなかった
- **WORK_LOG_2026-05-04.md記録：** PC画面 53→63 と記録あり
- **修正案：** 内訳を実態に合わせて修正（例：`PC 58 / HT 5 / Mobile 3 / Portal 9` で合計75）
  - ※ 正確なPC画面数はJSデータの全件カウントで要確認

#### 長期的な改善提案

**E. 最終更新日の自動化（将来対応）**

現在、最終更新日は手動で記載されているため更新漏れが発生しやすい。Git pre-push hookやGitHub Actionsを使って、コミット時に自動更新する仕組みを検討する価値がある。

**F. CLAUDE.mdとindex.htmlの二重管理解消（将来対応）**

工程進捗の状態がCLAUDE.mdとindex.htmlの2箇所に記録されており、今回のように不一致が発生しやすい。どちらか一方を正の情報源として、もう一方は参照のみにする運用ルールを設けることを推奨。
